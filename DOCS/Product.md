# CareerOS Product Vision

## Vision

CareerOS is a personal operating system for software engineers.

Its purpose is to help professionals manage every aspect of their career in one place.

CareerOS is intentionally broader than an interview tracker.

---

# Product Goals

CareerOS should help users:

* Track companies
* Manage applications
* Prepare for interviews
* Build professional relationships
* Capture networking context
* Organize career knowledge
* Learn from previous experiences

Eventually, the application should become an intelligent career companion.

---

# Target Audience

Primary audience:

Software engineers actively managing their careers.

Examples:

* Job seekers
* Senior engineers
* Engineering managers
* Students preparing for interviews

The initial development is optimized around the workflow of an experienced iOS engineer.

---

# Core Product Philosophy

CareerOS is not:

* A CRM
* A note-taking application
* A contacts application
* A task manager

Instead it connects information across domains.

Example:

Company → Application → Interview → Feedback

Person → Event → Interaction → Company

These relationships create valuable context over time.

# Product Modules

## Companies

Track organizations.

Examples:

* Uber
* Apple
* Salesforce

Future relationships:

* Applications
* People
* Notes
* Events

---

## Applications

Track job applications.

Information:

* Role
* Status
* Salary
* Source
* Dates

---

## Interviews

Owned by Applications.

Track:

* Interview rounds
* Feedback
* Outcome
* Preparation notes

---

## Professional Network

Track people met during professional interactions.

Examples:

* Recruiters
* Hiring managers
* Engineers
* Referrals

The focus is remembering context, not simply storing contact details.

---

## Events

Professional events.

Examples:

* Swift Bengaluru
* Uber Mobile Conference
* WWDC Watch Party

Events connect people, companies, and interactions.

---

## Interactions

Capture professional conversations.

Examples:

* Topics discussed
* Promises
* Follow-up actions
* Career advice

These become searchable historical context.

---

## Preparation

Track interview preparation.

Examples:

* Swift
* SwiftUI
* Concurrency
* System Design
* Architecture

Eventually include:

* Question bank
* Weakness tracking
* Study history

---

## Insights

Generate useful summaries.

Examples:

* Most applied company
* Most common rejection reason
* Strongest preparation areas
* Networking activity

Initially these will be rule-based.

AI-powered insights can be introduced later.

---

# Product Principles

Every feature should answer one or more of these questions:

Who?

* Person

Where?

* Company
* Event

What?

* Application
* Interview
* Question

Why?

* Interaction
* Feedback
* Notes

CareerOS should help users remember not only what happened, but why it mattered.

---

# Success Criteria

A successful CareerOS should allow a user to:

* Track career progress over years.
* Build institutional memory for professional relationships.
* Learn from previous interviews.
* Preserve networking context.
* Make better career decisions using accumulated knowledge.

Every new feature should contribute toward this vision.
